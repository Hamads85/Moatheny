# ADR-004: Magnetic Declination Calculator

**الحالة:** Accepted  
**التاريخ:** 30 يناير 2026  
**المؤلفون:** Knowledge Engineer

---

## السياق

البوصلة المغناطيسية تشير إلى **الشمال المغناطيسي** وليس **الشمال الحقيقي** (Geographic North). الفرق بينهما يسمى **الانحراف المغناطيسي (Magnetic Declination)**.

الانحراف المغناطيسي:
- يختلف حسب الموقع الجغرافي
- يتغير مع الوقت (بمعدل ~0.1-0.2° سنوياً)
- يمكن أن يصل إلى ±30° في بعض المناطق

بدون تعويض الانحراف، البوصلة قد تشير بزاوية خطأ تصل إلى 30° عن الاتجاه الصحيح.

## القرار

نقرر تطبيق **Magnetic Declination Calculator** لحساب وتعويض الانحراف المغناطيسي بناءً على الموقع الجغرافي.

### المميزات الرئيسية:
- **حساب دقيق**: استخدام World Magnetic Model (WMM) المبسط
- **تعويض تلقائي**: تحويل المغناطيسي إلى الحقيقي تلقائياً
- **Cache**: تخزين مؤقت للنتائج لتقليل الحسابات
- **دعم API خارجي**: إمكانية استخدام API خارجي للدقة العالية

## البدائل المدروسة

### البديل 1: تجاهل الانحراف المغناطيسي
**السلبيات:**
- أخطاء كبيرة في بعض المناطق (حتى 30°)
- عدم دقة في تحديد اتجاه القبلة
- تجربة مستخدم سيئة

**الإيجابيات:**
- بسيط
- لا يحتاج حسابات

**القرار:** رُفض - غير مقبول للدقة المطلوبة

### البديل 2: جدول ثابت للانحراف
**السلبيات:**
- غير دقيق (الانحراف يتغير حسب الموقع والزمن)
- يحتاج تحديث دوري
- غير عملي للعالم كله

**الإيجابيات:**
- بسيط
- سريع

**القرار:** رُفض - غير دقيق

### البديل 3: حساب محلي باستخدام WMM ✅
**السلبيات:**
- يحتاج تطبيق نموذج رياضي معقد
- دقة أقل قليلاً من WMM الكامل (±1-2° بدلاً من ±0.5°)

**الإيجابيات:**
- يعمل بدون إنترنت
- دقة جيدة (±1-2°)
- سريع بعد Cache

**القرار:** ✅ **مقبول** - أفضل توازن بين الدقة والاستقلالية

### البديل 4: API خارجي فقط
**السلبيات:**
- يحتاج إنترنت
- تأخير في الاستجابة
- تكلفة API calls
- عدم العمل بدون إنترنت

**الإيجابيات:**
- دقة عالية جداً (±0.5°)
- لا يحتاج تطبيق النموذج محلياً

**القرار:** رُفض - غير عملي، يُستخدم كخيار اختياري

## العواقب

### الإيجابية
- ✅ **دقة محسنة**: تقليل الأخطاء من ±30° إلى ±1-2°
- ✅ **تعويض تلقائي**: لا يحتاج تدخل المستخدم
- ✅ **يعمل بدون إنترنت**: حساب محلي
- ✅ **Cache**: أداء جيد بعد التخزين المؤقت
- ✅ **مرونة**: إمكانية استخدام API خارجي للدقة العالية

### السلبية
- ⚠️ **تعقيد**: نموذج رياضي معقد نسبياً
- ⚠️ **دقة محدودة**: ±1-2° بدلاً من ±0.5° (مقبول للبوصلة)
- ⚠️ **حسابات إضافية**: استهلاك CPU إضافي (ضئيل)
- ⚠️ **تحديث النموذج**: يحتاج تحديث معاملات WMM دورياً

### المحايدة
- 📊 **معلومات إضافية**: توفير معلومات عن الانحراف للمستخدم
- 🔧 **قابل للتخصيص**: إمكانية اختيار استخدام API خارجي

## التنفيذ

### النموذج المستخدم:
- **World Magnetic Model (WMM) المبسط**: نموذج Dipole مبسط مع تصحيح إضافي
- **معاملات WMM2020**: معاملات محدثة لعام 2020 مع حساب التغير السنوي

### الصيغة:
```swift
declination = atan2(Y, X)
```
حيث X و Y هي مركبات المجال المغناطيسي الأفقي

### المعاملات:
```swift
// معاملات WMM2020 (مبسطة)
g10: -29404.5 nT  // المجال الرئيسي
g11: -1450.7 nT
h11: 4652.5 nT

// التغير السنوي
g10Dot: 8.0 nT/year
g11Dot: 10.7 nT/year
h11Dot: -25.9 nT/year
```

## الملاحظات

- النموذج المبسط دقيق بما يكفي للبوصلة (±1-2°)
- للحصول على دقة أعلى (±0.5°)، يمكن تطبيق WMM الكامل أو استخدام API خارجي
- معاملات WMM يتم تحديثها كل 5 سنوات من NOAA
- Cache يقلل الحسابات بشكل كبير (نفس الموقع)

## التحسينات المستقبلية

- تطبيق WMM الكامل (12th order) للدقة العالية
- دعم IGRF (International Geomagnetic Reference Field)
- تحديث تلقائي لمعاملات WMM من NOAA
